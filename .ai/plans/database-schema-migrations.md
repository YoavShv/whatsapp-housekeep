# Feature: Database Schema & Migrations (Drizzle)

> **To implement:** Run `/execute .ai/plans/database-schema-migrations.md`

## Feature Description

Define the full Drizzle ORM schema for the WhatsApp building-management platform: five tables (ManagementCompany, Building, Resident, Complaint, Message), four TypeScript enum types, Drizzle relations, a typed db client exported from `lib/db/index.ts`, and a seed script that inserts demo data so the rest of the MVP demo can run without the consent flow.

## User Story

As a developer building the complaint-intake pipeline,
I want a typed database layer with schema, migrations, and seed data,
So that I can write repository/service code against real, versioned SQL tables.

## Design Summary

SQLite via `@libsql/client` + `drizzle-orm/libsql`. Enums are TypeScript `as const` arrays with inferred union types (SQLite has no native enum type). IDs are `text` UUIDs, timestamps are `integer({ mode: 'timestamp' })` with `sql\`(unixepoch())\`` defaults. `lib/db/index.ts` replaces `lib/db/client.ts` as the canonical db export, passing the schema to drizzle for the relational query API. `lib/db/client.ts` is left unchanged (nothing imports it). Migrations are generated by `drizzle-kit generate` and applied by `drizzle-kit migrate`.

### Patterns to Follow

**TABLE DEFINITION PATTERN** (Drizzle 0.45.x SQLite):
```typescript
// SOURCE: drizzle-orm/sqlite-core — follow exactly
import { sqliteTable, text, integer } from 'drizzle-orm/sqlite-core'
import { relations, sql } from 'drizzle-orm'

export const things = sqliteTable('things', {
  id: text('id').primaryKey(),
  name: text('name').notNull(),
  createdAt: integer('created_at', { mode: 'timestamp' })
    .notNull()
    .default(sql`(unixepoch())`),
})
```

**ENUM PATTERN** (SQLite has no native enum; use TS const arrays):
```typescript
export const categories = ['plumbing', 'electrical', 'elevator', 'cleaning', 'noise', 'parking', 'other'] as const
export type Category = typeof categories[number]

// In table: text('category').$type<Category>().notNull()
```

**SELF-REFERENCING FK PATTERN** (required for `dedupeTargetId` on complaints):
```typescript
import type { AnySQLiteColumn } from 'drizzle-orm/sqlite-core'

// In table:
dedupeTargetId: text('dedupe_target_id').references((): AnySQLiteColumn => complaints.id),
```

**DB CLIENT PATTERN WITH SCHEMA** (enables relational query API):
```typescript
// SOURCE: mirrors lib/db/client.ts:1-8 + adds schema
import { drizzle } from 'drizzle-orm/libsql'
import { createClient } from '@libsql/client'
import * as schema from './schema'

const client = createClient({ url: process.env.DATABASE_URL ?? 'file:./data/dev.db' })
export const db = drizzle(client, { schema })
```

**MIGRATE IN TEST PATTERN**:
```typescript
import { migrate } from 'drizzle-orm/libsql/migrator'

// In beforeAll:
const client = createClient({ url: ':memory:' })
const db = drizzle(client, { schema })
await migrate(db, { migrationsFolder: './db/migrations' })
```

**TYPE EXPORT PATTERN**:
```typescript
export type Thing = typeof things.$inferSelect
export type NewThing = typeof things.$inferInsert
```

```typescript
import { relations, sql } from 'drizzle-orm'
import type { AnySQLiteColumn } from 'drizzle-orm/sqlite-core'

// ── Enums ────────────────────────────────────────────────────────────────────

export const complaintCategories = [
  'plumbing', 'electrical', 'elevator', 'cleaning', 'noise', 'parking', 'other',
] as const
export type ComplaintCategory = typeof complaintCategories[number]

export const urgencyLevels = ['low', 'medium', 'high'] as const
export type Urgency = typeof urgencyLevels[number]

export const complaintStatuses = ['open', 'resolved'] as const
export type ComplaintStatus = typeof complaintStatuses[number]

export const messageSources = ['whatsapp', 'export'] as const
export type MessageSource = typeof messageSources[number]

// ── Tables ───────────────────────────────────────────────────────────────────

export const managementCompanies = sqliteTable('management_companies', {
  id: text('id').primaryKey(),
  name: text('name').notNull(),
  createdAt: integer('created_at', { mode: 'timestamp' })
    .notNull()
    .default(sql`(unixepoch())`),
})

export const buildings = sqliteTable('buildings', {
  id: text('id').primaryKey(),
  name: text('name').notNull(),
  address: text('address').notNull(),
  managementCompanyId: text('management_company_id')
    .notNull()
    .references(() => managementCompanies.id),
  createdAt: integer('created_at', { mode: 'timestamp' })
    .notNull()
    .default(sql`(unixepoch())`),
})

export const residents = sqliteTable('residents', {
  id: text('id').primaryKey(),
  phone: text('phone').notNull().unique(),
  name: text('name'),
  buildingId: text('building_id')
    .notNull()
    .references(() => buildings.id),
  consentedAt: integer('consented_at', { mode: 'timestamp' }),
  consentToken: text('consent_token').notNull(),
  createdAt: integer('created_at', { mode: 'timestamp' })
    .notNull()
    .default(sql`(unixepoch())`),
})

export const complaints = sqliteTable('complaints', {
  id: text('id').primaryKey(),
  buildingId: text('building_id')
    .notNull()
    .references(() => buildings.id),
  residentId: text('resident_id').references(() => residents.id),
  title: text('title'),
  category: text('category').$type<ComplaintCategory>().notNull(),
  urgency: text('urgency').$type<Urgency>().notNull().default('medium'),
  status: text('status').$type<ComplaintStatus>().notNull().default('open'),
  priority: integer('priority').notNull().default(0),
  dedupeTargetId: text('dedupe_target_id').references(
    (): AnySQLiteColumn => complaints.id,
  ),
  openedAt: integer('opened_at', { mode: 'timestamp' })
    .notNull()
    .default(sql`(unixepoch())`),
  resolvedAt: integer('resolved_at', { mode: 'timestamp' }),
})

export const messages = sqliteTable('messages', {
  id: text('id').primaryKey(),
  complaintId: text('complaint_id').references(() => complaints.id),
  buildingId: text('building_id')
    .notNull()
    .references(() => buildings.id),
  residentId: text('resident_id').references(() => residents.id),
  content: text('content').notNull(),
  source: text('source').$type<MessageSource>().notNull(),
  sentAt: integer('sent_at', { mode: 'timestamp' }).notNull(),
  createdAt: integer('created_at', { mode: 'timestamp' })
    .notNull()
    .default(sql`(unixepoch())`),
})

// ── Relations ────────────────────────────────────────────────────────────────

export const managementCompaniesRelations = relations(managementCompanies, ({ many }) => ({
  buildings: many(buildings),
}))

export const buildingsRelations = relations(buildings, ({ one, many }) => ({
  managementCompany: one(managementCompanies, {
    fields: [buildings.managementCompanyId],
    references: [managementCompanies.id],
  }),
  residents: many(residents),
  complaints: many(complaints),
  messages: many(messages),
}))

export const residentsRelations = relations(residents, ({ one, many }) => ({
  building: one(buildings, {
    fields: [residents.buildingId],
    references: [buildings.id],
  }),
  complaints: many(complaints),
  messages: many(messages),
}))

export const complaintsRelations = relations(complaints, ({ one, many }) => ({
  building: one(buildings, {
    fields: [complaints.buildingId],
    references: [buildings.id],
  }),
  resident: one(residents, {
    fields: [complaints.residentId],
    references: [residents.id],
  }),
  messages: many(messages),
  dedupeTarget: one(complaints, {
    fields: [complaints.dedupeTargetId],
    references: [complaints.id],
    relationName: 'dedupe',
  }),
  duplicates: many(complaints, { relationName: 'dedupe' }),
}))

export const messagesRelations = relations(messages, ({ one }) => ({
  complaint: one(complaints, {
    fields: [messages.complaintId],
    references: [complaints.id],
  }),
  building: one(buildings, {
    fields: [messages.buildingId],
    references: [buildings.id],
  }),
  resident: one(residents, {
    fields: [messages.residentId],
    references: [residents.id],
  }),
}))

// ── Inferred Types ───────────────────────────────────────────────────────────

export type ManagementCompany = typeof managementCompanies.$inferSelect
export type NewManagementCompany = typeof managementCompanies.$inferInsert

export type NewBuilding = typeof buildings.$inferInsert

export type NewResident = typeof residents.$inferInsert
export type Complaint = typeof complaints.$inferSelect
export type NewComplaint = typeof complaints.$inferInsert
```

```typescript
import { createClient } from '@libsql/client'
import { drizzle } from 'drizzle-orm/libsql'
import { migrate } from 'drizzle-orm/libsql/migrator'
import * as schema from '../lib/db/schema'
import { managementCompanies, buildings, residents } from '../lib/db/schema'

const COMPANY_ID = 'company-001'
const BUILDING_ID = process.env.POC_BUILDING_ID ?? 'building-001'
const RESIDENT_ID = 'resident-001'
const DEMO_PHONE = process.env.POC_RESIDENT_PHONE ?? '+972500000001'

async function seed() {
  const client = createClient({
    url: process.env.DATABASE_URL ?? 'file:./data/dev.db',
  })
  const db = drizzle(client, { schema })

  await migrate(db, { migrationsFolder: './db/migrations' })

  await db
    .insert(managementCompanies)
    .values({ id: COMPANY_ID, name: process.env.POC_COMPANY_NAME ?? 'Demo Management Co.' })
    .onConflictDoNothing()

  await db
    .insert(buildings)
    .values({
      id: BUILDING_ID,
      name: process.env.POC_BUILDING_NAME ?? 'Demo Building',
      address: '1 Demo St, Tel Aviv',
      managementCompanyId: COMPANY_ID,
    })
    .onConflictDoNothing()

  await db
    .insert(residents)
    .values({
      id: RESIDENT_ID,
      phone: DEMO_PHONE,
      name: 'Demo Resident',
      buildingId: BUILDING_ID,
      consentedAt: new Date(),
      consentToken: 'demo-consent-token',
    })
    .onConflictDoNothing()

  console.log(`Seed complete — building: ${BUILDING_ID}, resident phone: ${DEMO_PHONE}`)
  process.exit(0)
}

seed().catch((err) => {
  console.error('Seed failed:', err)
  process.exit(1)
})
```

## NOTES

**`lib/db/client.ts` vs `lib/db/index.ts`**: The scaffold (issue #1) created `client.ts`; the issue spec names `index.ts` as the key file. Both export `db`, but `index.ts` passes `{ schema }` to drizzle enabling the relational query API (`db.query.complaints.findMany(...)`). `client.ts` is left unchanged; it has no importers and will be removed in a future cleanup PR.

**SQLite enums**: SQLite has no `ENUM` type. The approach here is `text().$(type<>())` for runtime-safe type casting, with `as const` arrays as the single source of truth for valid values. Check constraints are intentionally omitted to stay lean; validation belongs at the service layer.

**Self-referencing FK** (`dedupeTargetId`): Drizzle requires the lazy-evaluated `(): AnySQLiteColumn => complaints.id` form for self-references to avoid hoisting issues.

**Timestamp representation**: `integer({ mode: 'timestamp' })` stores seconds since epoch; Drizzle maps to/from JS `Date`. The `sql\`(unixepoch())\`` default avoids any potential `defaultNow()` ambiguity on SQLite.

**`data/` directory**: Gitignored. The `db:migrate` and `db:seed` scripts handle a missing `data/` gracefully via `mkdir -p data &&` (noted in Task 5, Step 3). The seed script does not create the directory — `@libsql/client` creates the file but NOT its parent directory if it doesn't exist; the `mkdir -p data` before `db:migrate` covers this.

**Seed idempotency**: `.onConflictDoNothing()` means re-running the seed after the DB is already seeded is safe.
